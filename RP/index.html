1. User Authentication

60% Implemented

  • Completed
    3.1 Sign Up / Registration API
    API endpoints for email, phone, and standard profile registration values.
  • Completed
    3.1 Secure Login via Password
    Secure email and password verification with secure hashing algorithms.
  • Completed
    3.1 Secure Login via OTP
    OTP dispatch integrations via BulkGate SMS gateway and server-side verification pipelines.
  • Completed
    3.1 Google Social Login Handler
    Google authentication handlers validating ID tokens and generating corresponding JWT keys.
  • Pending
    3.1 Password Reset Trigger
    Authentication handlers to request forgot password tokens and securely update credentials in the database.
  • Pending
    3.9.8 Designation-Based Authorization Middleware
    Database roles and middleware security checks to differentiate Super Admin, Store Manager, Owner, and Team permissions.
  • Pending
    3.1 Multiple Delivery Address Storage
    Backend DB schema modifications to store and load multiple saved addresses per customer user.

90% Implemented

  • Completed
    3.2 Department Management API
    Hierarchical product department structures supporting order indexes and DisplayOrder variables.
  • Completed
    3.2 Sub-Department Classification
    Grouping and sorting mechanism for menu items under relevant categorization tabs.
  • Completed
    3.2 Customizations Database Relations
    Database schema and APIs mapping crusts, sizes, toppings, and addons to main items.
  • Completed
    3.2 Add-on Quantity Limits Constraint
    Enforcing DefaultQuantity and MaxQuantity limits (maximum 11 total addons constraint) within the CQRS pipeline.
  • Completed
    3.2 Deals & Promo Codes Storage
    CQRS execution steps to configure global deals, combos, and coupon records in Catalog Db.

    3.2 Menu Search and Filters API
    Searching and filtering backend parameters supporting veg/non-veg status, price ranges, and names.
  • Pending
    3.9.3 Store-Level Deal Overrides

    3.4 Checkout Order Generation

  • Completed
    3.8 Feedback Rating Storage API
    Ingestion endpoints receiving food, speed, and service ratings and linking them to order objects.
  • Pending

    3.4 Australian Postal Code Validation Logic

  • Pending
    3.4 Predefined Tip Value Integration

    3.4 Holiday and Weekend Surcharges
    Auto-calculation engine applying holiday/weekend surcharges dynamically based on store calendar parameters.
  • Pending
    3.5 COD Cash on Delivery State Modification
    Restructuring tracking events to omit 'Payment Successful' stage transitions when COD is chosen.
  • Pending
    3.5 GPS Real-Time Coordinate Ingestion
